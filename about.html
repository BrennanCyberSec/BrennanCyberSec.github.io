About This Lab

A personal cybersecurity lab for practical security research and skill development

The Mission

This lab serves as my personal portfolio and testing ground for cybersecurity research, purple teaming exercises, and continuous learning. It's designed to bridge the gap between theoretical knowledge and real-world security operations.

Every technique, tool, and scenario documented here comes from hands-on experience in building, breaking, and defending systems. The goal is simple: learn by doing, document the process, and share knowledge with the security community.

What Makes This Different

Purple Team Focus

Rather than focusing solely on offense or defense, this lab emphasizes the purple team approach where offensive techniques directly inform defensive improvements. Every attack is paired with detection strategies.

Automated Everything

Built on Ludus, the lab can spin up complex Active Directory environments, vulnerable machines, and security tools in minutes. Infrastructure as code means experiments are repeatable and documented.

Real-World Scenarios

Labs mirror actual enterprise environments and attack scenarios found in the wild. No CTF-style artificial puzzles - just practical techniques that matter in production environments.

The Tech Stack

This lab runs on a combination of open-source tools and industry-standard platforms:

  • Infrastructure: Ludus on Proxmox for automated range deployments
  • Offensive: Kali Purple, custom malware dev environment, various C2 frameworks
  • Defensive: Elastic Security, Velociraptor, Wazuh, Suricata
  • Analysis: REMnux, Wireshark, custom analysis scripts
  • Automation: Ansible, Packer, custom orchestration scripts

Learning Philosophy

Security isn't learned from reading alone. It requires breaking things, fixing them, and understanding why they broke in the first place. This lab embodies that philosophy through:

  • Hands-on exercises that build muscle memory
  • Real tools and techniques used by actual threat actors
  • Detection engineering that follows MITRE ATT&CK
  • Documentation that explains both the how and the why
