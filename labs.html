Lab Documentation Structure

Each lab will include:

  • Objective: What the lab aims to demonstrate or teach
  • Environment Setup: Tools, systems, and configurations used
  • Methodology: Step-by-step process and approach
  • Findings: Observations, data collected, and analysis
  • Detection: How attacks were detected and alerts generated
  • Key Takeaways: Lessons learned and practical applications

Planned Lab Topics

SIEM Detection Engineering

Planned

Building custom detection rules in ELK Stack for common attack techniques, correlated with MITRE ATT&CK framework. Focus on reducing false positives and improving alert fidelity.

Network Traffic Analysis

Planned

Analyzing malicious network traffic using Wireshark, Zeek, and Suricata to identify C2 communications, data exfiltration, and reconnaissance activities.

Incident Response Workflow

Planned

Practicing complete incident response procedures from initial detection through containment, eradication, and recovery. Focus on documentation and communication.

Active Directory Attack & Defense

Planned

Purple team exercise covering common AD attacks (Kerberoasting, AS-REP Roasting, Pass-the-Hash) and their detection signatures across multiple monitoring layers.

Endpoint Detection & Response

Planned

Analyzing endpoint telemetry from Sysmon and Windows Event Logs to detect malicious process behavior, persistence mechanisms, and privilege escalation attempts.

Phishing Analysis & Response

Planned

Deep-dive analysis of phishing emails including header analysis, URL/attachment inspection, and coordinated response procedures to contain potential compromises.
