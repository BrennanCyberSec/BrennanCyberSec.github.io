üîß Elasticsearch Startup Failure - Summary

Problem:

After editing elasticsearch.yml to uncomment and set network.host, http.port, cluster.name, and node.name, Elasticsearch failed to start with status=70/SOFTWARE error.

Root Causes:
  • Missing discovery setting - Setting network.host to an actual IP (instead of localhost) triggers Elasticsearch's production mode, which requires explicit discovery configuration.
  • Duplicate/conflicting entries - The config file had multiple duplicate lines appended to the bottom:
    • Two cluster.initial_master_nodes entries with different values
    • Two discovery.type: single-node entries
    • Conflicting http.host: 0.0.0.0
Solution:
  • Removed all cluster.initial_master_nodes lines
  • Removed http.host: 0.0.0.0
  • Kept a single discovery.type: single-node entry
Commands to Fix
sed -i '/cluster.initial_master_nodes/d' /etc/elasticsearch/elasticsearch.yml sed -i '/http.host: 0.0.0.0/d' /etc/elasticsearch/elasticsearch.yml
Lesson Learned:

When editing YAML configs, edit existing lines in place rather than appending new lines to the bottom, which can create duplicates that cause parse errors.
