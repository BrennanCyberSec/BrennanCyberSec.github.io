Creating the Shuffle Workflow

Setting up the webhook trigger and connecting Wazuh.

üîó

Create Webhook Trigger

Configure Shuffle to receive Wazuh alerts

In Shuffle, I created a new workflow and added a webhook as the trigger. This generates a unique URI that Wazuh will send alerts to.

Next, I edited the Wazuh Manager ossec.conf file to add a Shuffle integration that sends alerts in JSON format to the webhook. I filtered by a specific rule ID (100002) to only send relevant Mimikatz detection alerts.

ossec.conf Integration
<!-- Shuffle Integration --> <integration> <name>shuffle</name> <hook_url>https://shuffler.io/api/v1/hooks/webhook_[YOUR_ID]</hook_url> <rule_id>100002</rule_id> <alert_format>json</alert_format> </integration>

After restarting Wazuh Manager, I generated a Mimikatz event on my Windows VM to confirm the alert was produced and that Shuffle successfully received it.

02 // VIRUSTOTAL ENRICHMENT

Hash Extraction & Threat Intel

Extracting hashes and enriching with VirusTotal data.

üîç

Regex Capture Group

Extract SHA-256 hash from alert data

I added a regex capture step in Shuffle to extract the SHA-256 hash from the alert data. This regex looks into the hashes field, performs the pattern match, and pulls the hash value.

Regex Pattern
SHA256=([A-Fa-f0-9]{64})
ü¶†

VirusTotal Lookup

Query threat intelligence for the hash

I sent the extracted hash to VirusTotal for enrichment. After correcting the input to ensure only the hash value was passed (not the full field), the lookup succeeded with a 200 response.

üí°
Tip: Make sure to pass only the hash value to VirusTotal, not the entire field containing "SHA256=". Use the regex capture group output directly.
